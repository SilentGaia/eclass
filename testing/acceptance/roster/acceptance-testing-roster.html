CSC 309 User Acceptance Testing



Acceptance Test Plan For Roster by Eric Tran

The acceptance test plan is the login and roster of the E-Class.

Case No. User Inputs and Commands Expected Output Remarks                   Results                  
1 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. In the E-Class tool, user clicks once on the roster button in the taskbar. User sees roster interface display on the main E-Class window. Roster is initialized with initial roster of 7. Gene Fisher is top of list with green circle right of his name. Tests user login process and initialization of the roster. Pass.
2 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Open up roster by clicking on roster button in the taskbar. In the roster UI, user clicks on "Import" button positioned at the top. File chooser dialog pops up allowing for user to import an alternate roster. Tests E-Class feature of importing different rosters. Pass.
3 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Open up two process of the E-Class tool connected to the server. Log in as "Gene Fisher" and "password. Click on "roster" in taskbar. Log in as "Eric Tran" and "password" in other process. Roster UI updates with both Gene Fisher and Eric Tran logged in. Both have green circles to the right of their name. Tests the server communication between rosters of multiple processes. Tests the roster's update process. Pass.
4 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Open up two process of the E-Class tool connected to the server. Log in as "Gene Fisher" and "password" on one process. Click on "roster" in taskbar. Log in as "Eric Tran" and "password" in other process. On Fisher's process, click on the "assign" checkbox" of Eric Tran. Roster of Eric Tran updates with the ability to check other student's checkboxes. Tests the ability of users to assign other users the ability to assign. Pass.
5 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Log in as "Gene Fisher" and "password" on one process. Click on "roster" in taskbar in Fisher's process. Log in as "Eric Tran" and "password" in other process. Click on "roster" in taskbar in Tran's process. On Fisher's process, click on the "assign" checkbox of Eric Tran in roster. On Tran's process, click on the "assign" checkbox of Gene Fisher in roster. Roster of Gene Fisher still have the checkboxes of other student's enabled. Tests to make sure that instructor's will always have the ability to assign no matter what. Pass.
6 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Log in as "Gene Fisher" and "password". Click on "roster" button in the taskbar. Click on "roster" button in the taskbar again. Only one Roster UI displays in E-Class tool main window. Tests to make sure only one Roster is displayed in the E-Class tool. Pass.
7 Start server by typing "java -jar server.jar" in terminal or command prompt. Start E-Class by typing "java -jar classroom.jar" in terminal or command prompt. Log in as "Gene Fisher" and "password". Click on "roster" button in the taskbar. Click on the "x" button at the topright of the roster UI. Click on "roster" button in the taskbar again. Roster closes on the "x" button execution. Same roster opens again on the executon of the roster button. Tests to make sure only one Roster is displayed in the E-Class tool with the same roster. Pass.
