Electronic Classroom Tool Prototype



The Electronic Classroom Tool prototype is implemented as a pure GUI "store front" style, using Java Swing. "Store front" means the GUI displays sample interface screens, but there is no underlying functionality for operating on classroom data. The displayed screens are canned examples that illustrate typical data.

The prototype executable are in manifested jar file named classroom.jar. It can be launched by double clicking on its desktop icon. From the command line, it is executed using the "-jar" argument, i.e., " java -jar classroom.jar".

The top-level source files are in tha caltool package directory. The files are classroom/Main.java.

The package subdirectories with additional source are:

  • chat
  • classroom
  • drawingtools
  • filemanager
  • layers
  • presentation
  • roster
  • taskbar

Javadoc documentation for the prototype is in the ./.javadoc directory. There is a UNIX makefile for compiling the executable prototype and its Javadoc documentation.
